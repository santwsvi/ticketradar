# Reportando Vulnerabilidades de Segurança

**Não abra uma issue pública para vulnerabilidades de segurança.**

## Como Reportar

Envie um e-mail para: **privacidade@ticketradar.app**

Inclua:
- Descrição da vulnerabilidade
- Passos para reproduzir
- Impacto potencial
- Sugestão de correção (se tiver)

## Tempo de Resposta

- Confirmação de recebimento: **48 horas**
- Avaliação inicial: **7 dias**
- Correção (para issues críticas): **30 dias**

## Escopo

### Em escopo
- Injeção SQL / NoSQL
- XSS
- CSRF
- Exposição de dados pessoais (LGPD)
- Autenticação / autorização incorreta
- Dependências com CVEs críticos

### Fora de escopo
- Ataques de força bruta com rate limiting ativo
- Engenharia social
- Vulnerabilidades em serviços de terceiros (Twilio, Gmail, Railway)

## Divulgação Responsável

Pedimos que você:
1. Nos dê tempo razoável para corrigir antes de divulgar publicamente
2. Não acesse ou modifique dados de outros usuários
3. Não cause interrupção de serviço

Em troca, reconhecemos publicamente sua contribuição (se desejar).
